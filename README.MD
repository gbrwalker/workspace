# Workspace de Estudos e Projetos em Tecnologia

Este repositório centraliza meus estudos, exercícios práticos e projetos desenvolvidos ao longo da minha formação em tecnologia, com foco em **Ciência de Dados**, **Machine Learning** e **Desenvolvimento Web**.

A proposta do workspace é manter uma organização clara e evolutiva, permitindo acompanhar minha progressão técnica, desde fundamentos de programação até projetos aplicados e mais avançados.

---

## Estrutura do Repositório

### Cientista_de_Dados_EBAC
Conjunto de materiais desenvolvidos durante a formação de **Cientista de Dados pela EBAC**, organizados por módulos e temas.

Inclui:
- Fundamentos de Python para análise de dados
- Coleta, tratamento e preparação de dados
- Estatística aplicada
- Visualização de dados
- Modelagem estatística
- Machine Learning clássico
- Projetos práticos e estudos de caso

Cada pasta representa um módulo ou tema específico, contendo notebooks, scripts, datasets de apoio (quando aplicável) e documentações em Markdown.

---

### JavaScript_Projects
Projetos práticos em **JavaScript**, com foco em aprendizado de conceitos da linguagem, manipulação do DOM, eventos, animações e interações em front-end.

Inclui exercícios e mini-projetos independentes, organizados por pasta, cada um com seus arquivos HTML, CSS e JavaScript.

---

## Padrões e Boas Práticas

- O repositório mantém apenas código, notebooks e documentação.
- Arquivos grandes e dados sensíveis não são versionados.
- Configurações locais de IDE, ambientes virtuais e caches são ignorados via `.gitignore`.
- A branch principal (`main`) representa sempre o estado estável do workspace.

---

## Objetivo do Repositório

Este workspace tem como objetivos principais:
- Consolidar aprendizado técnico de forma organizada
- Servir como base de estudos contínuos
- Funcionar como portfólio técnico em evolução
- Facilitar futuras refatorações, melhorias e separação de projetos em repositórios dedicados

---

## Observação

Alguns projetos utilizam datasets externos ou arquivos grandes que não estão versionados no GitHub. Quando necessário, essas informações são descritas nos READMEs específicos de cada projeto.
